# SaS OS — Research → Decision → Build

Профессиональная система для поиска прибыльных ниш и быстрого запуска SaaS.

## Цель
Стабильно превращать сырой ресерч в:
1) валидированные идеи,
2) приоритетный build queue,
3) shipped MVP.

## Поток
1. `00_inbox/` — сырой вход (ссылки, заметки, скрины, логи)
2. `02_raw_notes/` — нормализованные заметки по источникам
3. `03_ideas/` — карточки идей (по шаблону)
4. `04_scored/` — скоринг + ранжирование
5. `07_decisions/` — решение: Kill / Hold / Build
6. `06_build_queue/` — что делаем на этой неделе
7. `08_kpis/` — результаты

## Стандарт качества
- Один факт = ссылка на источник.
- Любая идея без ICP + pain + willingness-to-pay → в Hold.
- Любая гипотеза без дедлайна проверки → недействительна.

## Быстрый старт
1. Сохрани входящие мысли в `00_inbox/inbox-YYYY-MM-DD.md`
2. Для каждой сильной находки создай карточку в `03_ideas/` из `templates/idea.md`
3. Прогони скоринг вручную по `templates/scoring.md` или скриптом `tools/score_ideas.py`
4. Перенеси топ-3 в `06_build_queue/week-YYYY-WW.md`
5. В конце недели зафиксируй решения в `07_decisions/decisions-YYYY-MM-DD.md`
